# 测评方法说明

本文档详细描述了 `proxy-review` 项目的测评指标、测试环境和评分标准，确保结果的客观性和可复现性。

---

## 1. 测评指标

### 1.1 连通率
- **首次成功率**：首次请求代理服务器返回有效响应的比例。
- **24小时成功率**：连续24小时内，每隔30分钟请求一次，成功响应的比例。
- **测试协议**：HTTP/HTTPS（暂不支持SOCKS5，视服务商情况而定）。
- **判定标准**：状态码200～299且响应体包含预期关键字（如 `httpbin.org` 返回 `origin` 字段）即为成功。

### 1.2 响应时间
- **平均延迟**：所有成功请求的响应时间平均值。
- **P95延迟**：95%的请求在此时间内完成。
- **P99延迟**：99%的请求在此时间内完成。

### 1.3 稳定性
- **IP存活率**：单个IP在30分钟内持续可用的比例。
- **掉线频率**：每24小时内IP断开重连的次数（适用于短效代理）。

### 1.4 IP纯净度
- **目标网站**：百度、淘宝、抖音等常见反爬网站。
- **判定**：返回正常页面内容（非验证码、封禁页）即为纯净。

### 1.5 地域分布
- **城市覆盖**：支持的城市列表。
- **运营商**：电信、联通、移动各自占比。

### 1.6 价格与性价比
- **计费方式**：按时间（日/月）、按IP数量、按流量等。
- **单位成本**：每个有效IP或每GB流量的平均价格。

### 1.7 API易用性
- **获取方式**：API接口、代理池、客户端软件。
- **文档质量**：完整性、示例代码、FAQ。
- **SDK支持**：官方提供的编程语言SDK。

---

## 2. 测试环境

- **服务器**：阿里云 ECS，华北2（北京）可用区A  
  - 实例规格：ecs.c6.large（2核4GB）  
  - 操作系统：Ubuntu 22.04 LTS  
  - 公网带宽：5Mbps  
- **并发**：5线程  
- **测试量**：每个服务商至少测试1000个独立IP（或持续24小时）  
- **测试工具**：自研 Python 脚本（[scripts/test_proxy.py](./scripts/test_proxy.py)）  
- **测试目标**：
  - `httpbin.org/ip`（验证代理IP）
  - `www.baidu.com`（真实网站连通性）
  - `www.taobao.com`（反爬检测）

---

## 3. 评分标准

每个维度的评分基于实际测试数据与行业平均水平的对比，最终推荐指数（1～5星）综合考量各项指标。

| 星级 | 含义 |
|------|------|
| ⭐⭐⭐⭐⭐ | 优秀，远超平均水平，无明显短板 |
| ⭐⭐⭐⭐ | 良好，主要指标达标，偶有小问题 |
| ⭐⭐⭐ | 合格，能满足基本需求，存在明显不足 |
| ⭐⭐ | 较差，频繁出现问题 |
| ⭐ | 极差，不推荐使用 |

---

## 4. 数据更新周期

- **月度更新**：每月第一周集中测试，第三周发布报告。
- **特殊更新**：若某服务商出现重大变化（如涨价、质量骤降），立即补充测试并更新。

---

## 5. 数据可复现性

所有原始测试数据（CSV格式）保存在 [data/](./data/) 目录中，包含时间戳、IP、响应状态、延迟等字段，可供第三方验证。
